Real NREL weather-station data · PVWatts physics model

Predicting rooftop solar output from real weather, not random numbers.

A neural network trained on genuine TMY3 station measurements for Sand Point, Alaska — irradiance, temperature, wind, humidity — with power output computed by pvlib's industry-standard PVWatts model, then validated on days it never saw during training.

What changed

Three fixes that make this a real ML pipeline

The original version generated inputs with np.random.uniform and a hand-written formula, then tested on the same rows it trained on. Here's what replaced that.

01 · DATA

Real station measurements

Hourly GHI, DNI, DHI, air temperature, wind speed, humidity and cloud cover from NREL's TMY3 dataset for a real station — not sampled from a uniform distribution.

02 · PHYSICS

PVWatts, not a guess

The power target comes from pvlib's solar-position, plane-of-array irradiance, cell-temperature and PVWatts inverter models — the same chain NREL and utilities use, not an invented formula.

03 · VALIDATION

Trained on the past, tested on the future

Data is split chronologically — train on the first 70% of the year, validate, then test only on the final unseen stretch. The scaler is fit on training data alone.

Training

Loss curve with early stopping

Honest limitations: the target variable is a physics-model estimate (PVWatts) driven by real weather inputs, not a utility-metered inverter reading — real installations also see soiling, shading, snow cover and inverter clipping that this model doesn't capture. Sand Point, Alaska is a high-latitude, low-irradiance site, so absolute watt values won't transfer to a sunnier location without retraining. Data source: NREL Typical Meteorological Year 3 (TMY3), distributed via the open-source pvlib library.
